The Problem

Traditional CI Workflow

  • Tests fail in CI → Human gets notified
  • Developer investigates manually
  • Hours or days to diagnose and fix
  • PR created, reviewed, merged manually
  • Re-run tests to verify — again manually

The Cost

  • Broken main branch blocks other PRs
  • Developer time wasted on flaky/simple bugs
  • Slow feedback loop (hours → days)
  • No guarantee the fix doesn't break other tests
  • Manual process is error-prone

What if the framework could detect, fix, and verify test failures autonomously?

Workflow 1: daily-test-fix.yml

Runs daily at midnight UTC or on manual dispatch. Handles both initial runs and reruns.

Job: run-tests

  • Checks out code, sets up Java 21
  • Installs Playwright browsers
  • On rerun: downloads testng-failed.xml from previous run
  • Runs only failed tests (rerun) or full suite (initial)
  • Uploads reports + failed XML as artifacts
  • Outputs tests_failed boolean

Job: open-issue-and-assign-copilot

  • Runs only IF tests failed
  • Downloads test report artifacts
  • Creates/updates GitHub issue with failure details
  • Labels: bug, auto-test-failure
  • Assigns Copilot agent (tries 5 bot names)
  • Fallback: @copilot mention in comment

Job: report-success

  • Runs only IF tests passed
  • On rerun: closes the auto-test-failure issue with success comment
  • On initial run: prints "No rerun required"
6

Workflow 2: auto-merge-copilot-pr.yml

Triggered when a human approves a Copilot-authored PR. Handles merge + rerun verification.

Step 1: Merge Approved Copilot PR

  • Validates PR author is a Copilot bot (copilot-swe-agent[bot], etc.)
  • Handles draft PRs → marks ready for review with polling (60s max)
  • Handles blocked state → enables auto-merge via GraphQL
  • Waits for mergeable state if unknown
  • Squash merges with descriptive commit message
  • Auto-closes the linked issue from PR body

Step 2: Trigger Test Rerun

  • Counts how many reruns have happened in the last 24 hours
  • Dispatches daily-test-fix.yml with incremented rerun_count
  • Maximum 2 reruns to prevent infinite loops
  • Run title shows "Rerun 1 — Verify Fix" or "Rerun 2 — Verify Fix"

Smart Rerun: Failed Tests Only

On reruns, only previously failed tests are executed using testng-failed.xml

How It Works

  • After test run, TestNG generates testng-failed.xml
  • Uploaded as artifact: testng-failed-xml
  • On rerun, workflow downloads this artifact from previous run
  • Uses GitHub CLI: gh run download
  • Maven Surefire runs with -DsuiteXmlFile=testng-failed.xml
  • Fallback: if XML not found, runs full suite

Benefits

  • Faster verification — skip passing tests
  • Focused feedback — only see relevant results
  • Resource efficient — less CI minutes
  • Clear signal — did the fix work for the broken tests?

pom.xml Config

<properties> <suiteXmlFile>testng.xml</suiteXmlFile> </properties> ... <suiteXmlFiles> <suiteXmlFile>${suiteXmlFile}</suiteXmlFile> </suiteXmlFiles>

Copilot Coding Agent Integration

Assignment Strategy

Tries 5 known bot names via REST API with agent_assignment payload:

const botNames = [ 'copilot-swe-agent', 'copilot-swe-agent[bot]', 'copilot', 'copilot[bot]', 'github-copilot[bot]' ];

If all fail → fallback @copilot mention in issue comment.

What the Agent Receives

  • Issue title: "Test failures detected"
  • Full failure stack traces from Surefire reports
  • Links to the failed GitHub Actions run
  • Rerun count (if applicable)
  • Labels: bug, auto-test-failure

What the Agent Does

  • Analyzes test failure logs
  • Reads source code to find root cause
  • Creates a fix PR targeting main
  • Includes Fixes #N in PR body

Reporting: ExtentReports + Screenshots

ExtentReportManager.java

  • Thread-safe: ThreadLocal<ExtentTest>
  • Timestamped HTML reports in reports/ directory
  • Captures system info: OS, Java version, browser, headless mode
  • Flushed on suite completion

TestListener.java Integration

  • onTestStart: Creates ExtentTest per method
  • onTestSuccess: Logs PASS with duration
  • onTestFailure: Logs FAIL + exception + Base64 screenshot
  • onTestSkipped: Logs SKIP
  • onFinish: Flushes report to HTML

Screenshot on Failure

When a test fails, a full-page screenshot is captured as Base64 and embedded directly into the ExtentReport HTML — no external file dependencies. Reports are uploaded as GitHub Actions artifacts for download.

Key Design Decisions

Safety Guardrails

  • Human-in-the-loop: Agent creates PR, human must approve
  • Max 2 reruns: Prevents infinite fix-fail loops
  • Squash merge: Clean git history
  • COPILOT_TOKEN secret: Scoped permissions

Reliability

  • Draft PR handling: markReadyForReview + polling
  • 5 bot name attempts: Handles API name changes
  • @copilot fallback: If REST assignment fails
  • Artifact fallback: Full suite if testng-failed.xml missing

Efficiency

  • Failed-only reruns: Via testng-failed.xml
  • Issue reuse: Updates existing issue vs creating new
  • Maven cache: Speeds up dependency resolution
  • Headless Chromium: No GUI overhead in CI

Observability

  • ExtentReports: Rich HTML with screenshots
  • Log4j2: Structured logging
  • Surefire reports: XML + TXT artifacts
  • Run naming: "Rerun 1 — Verify Fix"
